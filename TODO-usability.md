# MirrorDancer - TODO（ユーザビリティ評価・改善事項）

## 評価観点

### 1. ナビゲーション・情報設計

- [ ] 4タブ（比較・見本動画・自分の動画・設定）の役割分担が直感的か
- [ ] 比較タブ内の「比較モード ↔ 単体再生モード」切替が自然に使えるか
- [ ] 見本動画カードタップ → 比較画面、自分の動画行タップ → 単体再生、の導線が迷わないか
- [ ] 自分の動画行の「比較→」ボタンとタップ遷移の使い分けが理解できるか

### 2. 練習シーンでの実用性

- [ ] 汗をかいた手でのスワイプ操作の精度は十分か
- [ ] 動画の切り替えにかかるタップ数は最小限か
- [ ] 再生コントロールのボタンサイズと押しやすさは適切か
- [ ] 「踊る → 確認する → また踊る」のサイクルを少ないタップで回せるか
- [ ] スマホを床や棚に置いた状態で操作しやすいか

### 3. 同期・速度調整のわかりやすさ

- [ ] 開始位置合わせの ±0.1s / ±0.5s 操作がダンス初心者にも理解できるか
- [ ] 見本と自分で速度が個別にある意味が伝わるか
- [ ] 同時再生と個別再生の使い分けが混乱しないか
- [ ] 開始位置合わせのUI（数値ボタン方式）が最適か、タップ同期方式の方が良いか検討

### 4. データ管理の安全性

- [ ] スワイプで意図しない非表示・削除が発生しないか
- [ ] 見本削除時に紐づく自分の動画も消える点の警告が十分か
- [ ] 削除の確認ダイアログの表示タイミングと文言が適切か
- [ ] 非表示と削除の違いがユーザーに明確に伝わるか
- [ ] 非表示動画の復元導線（設定タブ）にたどり着けるか

### 5. 視認性・フィードバック

- [ ] 再生中かどうかが一目で把握できるか
- [ ] どの動画が選択中かの視覚的フィードバックが十分か
- [ ] 速度の現在値が見やすいか
- [ ] スワイプ操作のヒントテキストだけで初見ユーザーが気づけるか
- [ ] オンボーディング（初回チュートリアル）が必要か検討
- [ ] 離れた位置からでも再生状態がわかるか（ダンス練習中の距離）

### 6. スケーラビリティ

- [ ] 見本動画20〜30本での一覧の探しやすさ
- [ ] 自分の動画100本超での表示パフォーマンス
- [ ] 日付・見本別の切り替えと折りたたみだけで十分か
- [ ] 検索・フィルタ機能の必要性を検討
- [ ] 動画ファイルのストレージ容量管理の仕組みが必要か

### 7. アクセシビリティ

- [ ] 片手操作でどこまでできるか
- [ ] 左利きの場合のスワイプ方向（左→右）の自然さ
- [ ] 文字サイズの十分さ（小さい文字が多くないか）
- [ ] カラーコントラストは十分か（ダークテーマ前提）
- [ ] 横向き（ランドスケープ）対応の必要性

## 推奨する評価方法

### ユーザビリティテスト（推奨）

- 対象: 実際のダンス練習者 3〜5人
- 方式: タスクベーステスト
- タスク例:
  1. 「見本動画Aと自分の練習動画を並べて比較再生してください」
  2. 「自分の動画を0.5倍速で再生してください」
  3. 「見本動画を非表示にしてください」
  4. 「非表示にした動画を復元してください」
  5. 「新しく撮影した動画を追加してください」
- 観察ポイント: 完了までの時間、迷ったポイント、誤操作の回数
