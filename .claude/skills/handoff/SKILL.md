# チャット引き継ぎスキル

## 目的

現在の会話コンテキストを整理し、新しい Claude チャットにスムーズに引き継げるハンドオフ文書を生成する。

## 手順

### 1. 現状把握

以下を確認する:

- `docs/` フォルダの設計書が最新かチェック（古い場合は先に `/update-docs` の実行を推奨）
- `git status` / `git diff` で未コミットの変更を確認
- 現在進行中のタスクや未解決の問題を整理

### 2. ハンドオフ文書の生成

`docs/HANDOFF.md` を生成または更新する。以下の構成で書く:

```markdown
# MirrorDancer ハンドオフ文書
生成日時: YYYY-MM-DD

## プロジェクト概要
（1-2行で簡潔に）

## 設計書の読み方
docs/ フォルダの設計書一覧と読む順序を案内

## 現在の状態
- 最後にビルド/テストが通った状態か
- 未コミットの変更があるか
- 既知の問題・バグ

## 進行中のタスク
（引数 $ARGUMENTS の内容を反映）
- 何をしようとしていたか
- どこまで進んだか
- 次に何をすべきか

## 注意点・制約
- 技術的な注意点（compare_tab.dart が大きい、Cupertino のみ、など）
- ユーザーの好み・フィードバック
- 避けるべきこと

## 新しいチャットへの指示テンプレート
（そのままコピペで使えるプロンプト）
```

### 3. 指示テンプレートの生成

ハンドオフ文書の末尾に、新しいチャットに貼り付けるためのテンプレートを含める:

```
MirrorDancer という Flutter アプリの開発を続けます。

## セットアップ
1. まず docs/ フォルダの設計書を全て読んでください
2. docs/HANDOFF.md に現在の状況と次のタスクが書いてあります

## 次にやること
（具体的なタスク）

## 制約
- Cupertino ウィジェットのみ使用（Material は使わない）
- 日本語 UI
- ダークモードベース
```

### 4. 結果報告

- `docs/HANDOFF.md` のパスを表示
- 新しいチャットへのコピペ用テンプレートをそのまま出力
- 設計書が古い場合は `/update-docs` の実行を促す

## ルール

- 会話中に得た暗黙知（ユーザーの好み、過去のエラー対応など）も漏らさず記録する
- 技術的な判断の理由（なぜ C案タイムラインにしたか等）も含める
- テンプレートはそのままコピペで使える形にする
- 日本語で記述する
